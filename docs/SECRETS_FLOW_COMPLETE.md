# Application Secrets Management - Complete Flow

**Status:** ✅ COMPLETE - App reads from AWS Secrets Manager

---

## The Complete Architecture

### Local Development

```
1. User runs: python scripts/local-dev-setup-env.py
   ↓
2. Script decrypts vault.yml and generates .env
   ↓
3. App starts (app/__init__.py)
   - Detects .env exists
   - Loads .env via load_dotenv()
   ↓
4. App config (app/config.py)
   - Tries to reach AWS Secrets Manager
   - Falls back to environment variables (.env)
   ↓
5. App runs with local secrets from .env
```

### Production Deployment

```
1. Deployment playbook runs (setup.yml)
   ↓
2. Creates systemd service with environment variables:
   - SECRET_NAME={{ app_name }}/secrets
   - AWS_REGION={{ aws_region }}
   - EC2 instance configured with IAM role
   ↓
3. No .env file on server
   ↓
4. App starts (app/__init__.py)
   - Detects .env does NOT exist
   - Skips load_dotenv()
   ↓
5. App config (app/config.py)
   - Tries to reach AWS Secrets Manager
   - Uses environment variables (SECRET_NAME, AWS_REGION)
   - Fetches secrets from AWS Secrets Manager
   ↓
6. App runs with secrets from AWS Secrets Manager
```

## Code Flow

### Application Initialization (app/__init__.py)

```python
# Try to load .env (only if it exists)
env_path = Path(__file__).parent.parent / '.env'
if env_path.exists():
    load_dotenv(dotenv_path=env_path)  # Local dev
else:
    pass  # Production - use systemd env vars and AWS Secrets Manager
```

**Result:**
- ✅ Local dev: .env loaded into environment
- ✅ Production: .env not needed, skip gracefully

### Configuration (app/config.py)

```python
def get_secrets_from_aws():
    """Fetch secrets from AWS Secrets Manager."""
    secret_name = os.environ.get('SECRET_NAME', 'app-item-listing-tool/production')
    region_name = os.environ.get('AWS_REGION', 'us-east-1')
    
    try:
        client = boto3.client('secretsmanager', region_name=region_name)
        response = client.get_secret_value(SecretId=secret_name)
        return json.loads(response['SecretString'])
    except:
        # Fall back to environment variables if AWS unavailable
        return {}

_secrets = get_secrets_from_aws()

def get_secret(key, default=None):
    """Get secret from AWS or environment variable."""
    return _secrets.get(key) or os.environ.get(key) or default

class Config:
    SECRET_KEY = get_secret('SECRET_KEY', 'dev-secret-key')
    # ... other config ...
```

**Result:**
- ✅ Tries AWS Secrets Manager first
- ✅ Falls back to environment variables
- ✅ Falls back to defaults if both missing

### Systemd Service (deployment/files/app.service.j2)

```ini
[Service]
# ... existing config ...

# AWS Secrets Manager environment variables
Environment="SECRET_NAME={{ app_name }}/secrets"
Environment="AWS_REGION={{ aws_region }}"

# Optional .env file (commented out, not used in production)
# EnvironmentFile={{ app_dir }}/.env
```

**Result:**
- ✅ Sets SECRET_NAME for app to know which AWS secret to fetch
- ✅ Sets AWS_REGION for where to fetch from
- ✅ EC2 instance's IAM role provides AWS authentication
- ✅ No .env file needed

## What Gets Stored Where

### Local Development (.env)
```bash
# Generated by: python scripts/local-dev-setup-env.py
# Contains: Flask, eBay, User auth secrets
# Location: ./.env (in .gitignore)
# Usage: Flask app reads from here
```

### Deployment Vault (vault.yml)
```yaml
# Manual edit by user
# Contains: All secrets needed for deployment AND local dev
# Location: deployment/group_vars/vault.yml (encrypted, in git)
# Usage: Generates .env for local dev, syncs to AWS for production
```

### AWS Secrets Manager
```json
{
  "SECRET_KEY": "...",
  "FLASK_ENV": "production",
  "USERS": "...",
  "EBAY_ENVIRONMENT": "...",
  "EBAY_SANDBOX_APP_ID": "...",
  "EBAY_PRODUCTION_APP_ID": "...",
  "EBAY_VERIFICATION_TOKEN": "...",
  ...
}
```
- Created by: Deployment playbook syncs from vault.yml
- Usage: Production Flask app reads from here
- Authentication: EC2 IAM role (no credentials needed)

## How to Verify It Works

### Local Development
```bash
# Generate .env from vault
python scripts/local-dev-setup-env.py

# Verify .env was created
ls -la .env

# Run app
python -m app

# Check logs - should NOT see AWS errors (expected offline)
# Should see: "Warning: Secrets Manager not available, using environment variables"
```

### Production
```bash
# After deployment, SSH to server
ssh ec2-user@your-server

# Check systemd service
systemctl status rampe

# Verify environment variables are set
systemctl show -p Environment rampe
# Should show: SECRET_NAME=rampe/secrets, AWS_REGION=us-east-2

# Check app logs
journalctl -u rampe -f

# Should NOT see any "no .env file" errors
# Should NOT have any issues getting secrets

# Test secret access manually
curl http://localhost:8000  # App should be running
```

## Next Steps if Issues Occur

If secrets aren't loading, check:

1. **Local Development:**
   ```bash
   python scripts/local-dev-setup-env.py  # Regenerate .env
   cat .env | grep SECRET_KEY             # Verify secret is there
   python -m app                           # Try running app
   ```

2. **Production:**
   ```bash
   systemctl show -p Environment rampe    # Check env vars set
   aws secretsmanager get-secret-value \
     --secret-id rampe/secrets \
     --region us-east-2                   # Verify AWS has the secret
   journalctl -u rampe -f -n 50           # Check recent logs
   ```
